Boltzmann Machines - An Origin Story for Deep Generative Models

These days, deep learning is all the hype in machine learning. Spawning thousands of papers in artificial intelligence, we have countless deep models which, some may argue, make up the majority of "intelligent" software that runs our world today. But where did these networks come from? How did researchers come up with all of these fancy generative networks we have today such as variational autoencoders (VAEs) and generative adversarial networks (GANs)? In this post, we're going to explore a structurally simple network called the Restricted Boltzmann Machine (RBM) which is the epitome of the application of probablistic graphical models to deep learning.

Boltzmann Machines

First of all, what is a regular Boltzmann machine? Based on the stochastic Ising model for magnetic dipole moments, it is a network that is based on statistical mechanics. It is also known as a stochastic Hopfield network, where Hopfield networks are a type of recurrent neural network which has been known to provide a model for understanding human memory.

For machine learning purposes, however, what is important about the Boltzmann Machine is that it provides a way to learn arbitrary probability distributions over binary vectors. Specifically, a Boltzmann Machine contains a set of binary random variables that are connected to each other by a set of weights which signify how likely pairs of variables will take on the same value. Each variable also has a bias term which represents how likely a given variable will turn on. The term "Boltzmann" is in the name because the probability of a given configuration/state x\bm{x}x of a Boltzmann machine follows a Boltzmann Distribution:

p(x)=e−E(x)∑x′e−E(x′)p(\bm{x}) = \frac{e^{-E(\bm{x})}}{\sum_{\bm{x'}}e^{-E(\bm{x'})}} p(x)=∑x′​e−E(x′)e−E(x)​

where ppp denotes probability, EEE is the energy function of a given state x\bm{x}x of the machine, and x′\bm{x'}x′ represents every possible state of the machine. Let's draw an example configuration of a Boltzamann machine.


Figure 1: Sample Boltzmann Machine with three binary variables

The values connecting pairs of variables are weights while the values emanating from the variables are biases. The set of variables, weights, and biases makes up a configuration of the Boltzmann machine. The energy of the configuration is given as the negative sum of products of weights and pairs of variables and products of biases and individual variables. In the case of Figure 1, the energy would be calculated as:

E=−[(2x1x2+1x1x3+(−1)x2x3)+(2x1+3x2+4x3)]E=-[(2x_1x_2 + 1x_1x_3 + (-1)x_2x_3) + (2x_1 + 3x_2 + 4x_3)] E=−[(2x1​x2​+1x1​x3​+(−1)x2​x3​)+(2x1​+3x2​+4x3​)]

As seen above, the most probable state occurs when all variables are equal to 0. In the general case, given a state vector of ddd variables x∈Rd\bm{x} \in \mathbb{R}^dx∈Rd, weights matrix W∈Rd×d\bm{W} \in \mathbb{R}^{d\times d}W∈Rd×d, and biases vector b∈Rd\bm{b} \in \mathbb{R}^db∈Rd, the energy is given by:

E(x)=−xTWx−bTxE(\bm{x}) = -\bm{x}^TW\bm{x} - \bm{b}^T\bm{x} E(x)=−xTWx−bTx

Here, element WijW_{ij}Wij​ at the ithi^{\text{th}}ith row and jthj^{\text{th}}jth column of W\bm{W}W is the weight connecting variables xix_ixi​ and xjx_jxj​, and each element bkb_kbk​ of b\bm{b}b is the bias contributing to only xkx_kxk​. We can see that while a single energy is quite straightforward to compute for a given state (and hence the probability as well), the sum of all probabilities is not. Also known as the partition function, it is highly sensitive to the number of variables in the Boltzmann machine. Even only three variables has eight possible states, and as we increase the number of variables, the total number of states increases exponentially. This is a problem, especially since we live in a world of high-dimensional data that we would like to eventually learn.

Learning Boltzmann Machines

In order to learn a Boltzmann Machine, we must try to find a set of parameters (weights and biases) that maximizes p(x)p(\bm{x})p(x), also known as the likelihood of some given training data x(n)\bm{x^{(n)}}x(n), n∈{1,...,N}n \in \{1,...,N\}n∈{1,...,N} (Superscripts denote a particular training example, not dimension). To do this, we can apply a standard technique called gradient descent which changes the parameters based on the gradient of what we would like to maximize. Since the probability is given by a Boltzmann Distribution, we can take the natural logarithm, sum over all training examples, and then divide by the total number of examples to get the mean log likelihood. Using the form of p(x)p(\bf{x})p(x) given at the top of this post, the log likelihood is given as:

There are a couple important things to notice here about the positive statistics. One is that it is equivalent to an expected value over the given data, meaning it is relatively easy to compute since one would just need to take an average over a batch of samples. The other neat insight is that the derivatives only depend on the units that are connected to the parameter in question. In other words, one only needs to know the values of the variables next to the parameter to update it. What is most exciting about this is that this learning rule is very similar to how the biological brain has been hypothesized to learn. This is a form of Hebbian learning which can be summarized by the popular saying: "neurons that fire together, wire together". Since parameter updates of a Boltzmann Machine are local, some argue that this learning update can be perceived as "biologically plausible".

Let us now handle the negative statistics. This is a little more difficult to manipulate, but not too much of a problem if we use the chain rule:

Notice that the negative statistics look extremely similar to the positive statistics except for one crucial difference: the distribution we are sampling from. The total gradient of the log-likelihood is given by the deceptively simple formulas below:

∂L∂Wij=Ex∼data[xixj]−Ex∼model[xixj]∂L∂bi=Ex∼data[xi]−Ex∼model[xi]\begin{aligned} \frac{\partial L}{\partial W_{ij}} = \mathbb{E}_{\bm{x}\sim \text{data}}[x_ix_j] - \mathbb{E}_{\bm{x}\sim \text{model}}[x_ix_j] \\ \frac{\partial L}{\partial b_i} = \mathbb{E}_{\bm{x}\sim \text{data}}[x_i] - \mathbb{E}_{\bm{x}\sim \text{model}}[x_i] \end{aligned} ∂Wij​∂L​=Ex∼data​[xi​xj​]−Ex∼model​[xi​xj​]∂bi​∂L​=Ex∼data​[xi​]−Ex∼model​[xi​]​

While the positive statistics are relatively straightforward to compute since we can use the data directly, the negative statistics are not as easy to find. Since a Boltzmann machine can have a large number of states (recall the exponential growth in variables shown above), the partition function is typically intractable, meaning it is infeasible to sample from our model distribution p(x)p(\bm{x})p(x). Luckily for us, there is a convenient method to approximate the model distribution by using the conditional distribution.

Gibbs Sampling

Also known as a special case of the Metropolis-Hastings algorithm, Gibb's sampling aims to approximate an intractable distribution by using a friendlier conditional distribution. The algorithm can be summarized as follows:

  1. Start with an arbitrary x\bm{x}x.
  2. Set index i=1i=1i=1.
  3. Set xix_ixi​ to a sample taken from p(xi∣x−i)p(\bm{x_i}|\bm{x_{-i}})p(xi​∣x−i​) where x−i\bm{x_{-i}}x−i​ represents {xj∣j≠i}\{x_j | j\neq i\}{xj​∣j​=i}.
  4. Repeat Step 3 for all i,i∈{2,...,d}i, i \in \{2,...,d\}i,i∈{2,...,d}.
  5. Repeat Steps 2-4 kkk times.

Note that one of the hyperparameters of Gibb's sampling is kkk, i.e., the number of times we update all components of x\bm{x}x. In theory, the larger kkk is, the closer the sampled x\bm{x}x approximates p(x)p(\bm{x})p(x), i.e., the model.

So this seems straightforward to apply, but only if it easy to sample from the conditional distribution. Fortunately, since we are dealing with a Boltzmann Machine, it is!

We can now combine this conditional distribution with Gibbs sampling to yield our final learning algorithm for Boltzmann Machines.

Contrastive Divergence

Contrastive Divergence is the name of the algorithm, and it mainly combines the two sections above. It first samples a set of data and computes the positive statistics on it. After copying these samples into temporary variables, it then runs Gibbs sampling on the latter to compute the negative statistics. The gradient is set according to these statistics, allowing the parameters to be updated with some learning rate factor λ\lambdaλ. Actually, contrastive divergence is a very powerful algorithm that can be applied to learn most probabilistic models with intractable partition functions. A summary of the algorithm applied to Boltzmann Machines is given as below:

  1. Sample a batch of MMM data points {x(1),...,x(M)}\{\bm{x}^{(1)},..., \bm{x}^{(M)}\}{x(1),...,x(M)}.
  2. Set the gradients to the positive statistics: ∂L∂Wij←1M∑m=1Mxi(m)xj(m)\frac{\partial L}{\partial W_{ij}} \leftarrow \frac{1}{M}\sum_{m=1}^M x_i^{(m)}x_j^{(m)}∂Wij​∂L​←M1​∑m=1M​xi(m)​xj(m)​, ∂L∂bi←1M∑m=1Mxi(m)\frac{\partial L}{\partial b_i} \leftarrow \frac{1}{M}\sum_{m=1}^M x_i^{(m)}∂bi​∂L​←M1​∑m=1M​xi(m)​.
  3. Create copies of the data points from Step 1 as {x~(1),...,x~(M)}\{\tilde\bm{x}^{(1)},..., \tilde\bm{x}^{(M)}\}{x~(1),...,x~(M)}.
  4. For each copy, run Gibbs sampling by using the conditional distribution derived above. Practically, to sample from it, simply choose a random number from the uniform distribution and compare it to the value of the distribution. The final updated set of copies is now approximately distributed according to the model's distribution.
  5. Update the gradients with the negative statistics: ∂L∂Wij←∂L∂Wij−1M∑m=1Mx~i(m)x~j(m)\frac{\partial L}{\partial W_{ij}} \leftarrow \frac{\partial L}{\partial W_{ij}} - \frac{1}{M}\sum_{m=1}^M \tilde x_i^{(m)}\tilde x_j^{(m)}∂Wij​∂L​←∂Wij​∂L​−M1​∑m=1M​x~i(m)​x~j(m)​, ∂L∂bi←∂L∂bi−1M∑m=1Mx~i(m)\frac{\partial L}{\partial b_i} \leftarrow \frac{\partial L}{\partial b_i} - \frac{1}{M}\sum_{m=1}^M \tilde x_i^{(m)}∂bi​∂L​←∂bi​∂L​−M1​∑m=1M​x~i(m)​.
  6. Update the parameters: Wij←Wij+λ∂L∂WijW_{ij} \leftarrow W_{ij} + \lambda \frac{\partial L}{\partial W_{ij}}Wij​←Wij​+λ∂Wij​∂L​, bi←bi+λ∂L∂bib_i \leftarrow b_i + \lambda \frac{\partial L}{\partial b_i}bi​←bi​+λ∂bi​∂L​.
  7. Repeat steps 1-6 until some convergence criterion is met (e.g., the gradients do not change much or we reach the maximum number of iterations).

Some people like to say that contrastive divergence is loosely based on what the brain does. Updating the gradient with the positive statistics is said to be the "positive phase" of learning which corresponds to enforcing your knowledge of the world by directly observing the data around you, i.e., your environment. Updating the gradient with the negative statistics is called the "negative phase" of learning which corresponds to rejecting incorrect knowledge from the current model of the brain. This rejection would happen during the sleep/dream phase when the brain is unable to observe the environment and only has its current model to work off of. Although in machine learning, both positive and negative phases are typically done at the same time, this provides yet another neat parallel between Boltzmann Machines and the brain.

Restricted Boltzmann Machines

Now that we know how Boltzmann Machines work and how they can be learned, we have all the tools we need to understand a subset of these machines, the Restricted Boltzmann Machine (RBM). An RBM is simply a regular Boltzmann Machine with binary variables divided into two sets: a visible set and a hidden set. It is also a bitartite graph, meaning every visible variable is connected to a hidden one and every hidden variable is connected to a visible one. The machine is "restricted" because there are no connections between pairs of visible variables and no connections between pairs of hidden ones. In practice, the visible nodes represent actual data while the hidden nodes represent more abstract features that can be extracted from the data. An example of an RBM is given below:


Figure 2: Sample Restricted Boltzmann Machine with four visible variables and three hidden ones

As we can see, the RBM is a very simple architecture, but it also exemplifies several fundamental building blocks of deep graphical models. Firstly, it arranges the nodes into fully connected layers that are parameterized by a weight matrix W\bm{W}W and biases b\bm{b}b and c\bm{c}c. Secondly, it introduces the notion of learning a set of "latent" variables, i.e., other representations of the data that are not immediately seen in the data itself.

One of the first papers which introduced RBMs applied it to the Netflix dataset to predict user preferences for movies. The visible layer represented the user's ratings of individual movies, and the hidden layer represented more abstract features of the movies that likely influenced the users' decisions (e.g., genre, length, time of release, etc.). RBMs are quite versatile and can be applied to a large variety of situations. For example, in Figure 2, the four visible units could represent whether or not a student decides to show up to a lecture of one of their four courses, and the three hidden units could represent the student's aptitude, the lecturer's teaching ability, and the difficulty of the course.

Learning RBMs

To learn an RBM, we can leverage much of the results that were already found for regular Boltzmann Machines. However, now the weight matrix is constrained such that each element WijW_{ij}Wij​ is the weight connecting the ithi^{\text{th}}ith visible unit to the jthj^{\text{th}}jth hidden one. The derivatives are given below:

∂L∂Wij=Ev∼data[vihj]−Ev,h∼model[vihj]∂L∂bi=Ev∼data[vi]−Ev∼model[vi]∂L∂ci=Ev∼data[hi]−Ev,h∼model[hi]\begin{aligned} \frac{\partial L}{\partial W_{ij}} = \mathbb{E}_{\bm{v}\sim \text{data}}[v_ih_j] - \mathbb{E}_{\bm{v},\bm{h}\sim \text{model}}[v_ih_j] \\ \frac{\partial L}{\partial b_i} = \mathbb{E}_{\bm{v}\sim \text{data}}[v_i] - \mathbb{E}_{\bm{v}\sim \text{model}}[v_i] \\ \frac{\partial L}{\partial c_i} = \mathbb{E}_{\bm{v}\sim \text{data}}[h_i] - \mathbb{E}_{\bm{v},\bm{h}\sim \text{model}}[h_i] \end{aligned} ∂Wij​∂L​=Ev∼data​[vi​hj​]−Ev,h∼model​[vi​hj​]∂bi​∂L​=Ev∼data​[vi​]−Ev∼model​[vi​]∂ci​∂L​=Ev∼data​[hi​]−Ev,h∼model​[hi​]​

Again, the first terms of these differences are relatively easy to compute. The only small difference is that the hidden units must be sampled using the network's existing parameters given the visible units from the data. The second terms can be computed using Gibbs sampling, meaning we will need expressions for the conditional distributions of the RBM's units. In fact, since the RBM is restricted, we can see from Figure 2 that each layer is conditionally independent, i.e., given one layer, all units within the other layer are independent from each other. This observation allows us to run a more efficient version of Gibbs sampling called block Gibbs sampling shown below in Figure 3.


Figure 3: Visualization of block Gibbs sampling for the RBM in Figure 2

As done before for regular Boltzmann Machines, this chain of Gibbs sampling is computed for each sample of the training set, and the positive and negative statistics are taken from (v1,h1)(\bm{v}^1, \bm{h}^1)(v1,h1) and (vk,hk)(\bm{v}^k, \bm{h}^k)(vk,hk), respectively. Notice how we leverage conditional independence by sampling an entire layer at a time.

Now all that is left is to write down the conditional distributions for RBMs! This is again straightforward since we already derived the general case above, but now we can make use of the conditional independence of each layer by multiplying the individual conditional distributions for each unit within said layer. Let's assume that there are VVV visible units and HHH hidden ones.

We now have all the necessary tools to construct and train an RBM! But, as mentioned before, this is only the beginning of the rich history of deep generative models.

Going Deeper

There are several networks which directly build off the simple structure of the RBM. You may have noticed that the RBM by itself is not a deep network. However, what if we were to stack several of them on top of each other during the training process? We would then have what is called a Deep Belief Network (DBN), the first successful deep learning network which sparked the ubiquitous drive for deep learning we have today. An example DBN is shown in Figure 4 below:


Figure 4: Deep Belief Network

In a DBN, all connections between neighbouring layers are directed in the direction of the visible units, except for the top two hidden layers which are connected with undirected weights. This structure arises from the way a DBN is typically trained. If we have RRR hidden layers, the training process involves training RRR RBMs in a bottom-up approach. In other words, we start with the observed data in the visible layer, train the RBM between this and the first hidden layer using the usual techniques discussed above, and then repeat this process using the trained first hidden layer as the "visible" layer to train the second RBM. After this entire process is over, we end up with one trained RBM at the top and a directed generative model in the rest of the network. This combination of models is what makes up a DBN. It can be shown that, the more layers of RBMs that are trained, the more a variational lower bound on the log likelihood of the data increases, leading to a better approximation of p(x)p(\bm{x})p(x).

Although DBNs by themselves are interesting as a generative model, they were used more frequently as a pre-training phase for supervised learning. Basically, after training a DBN, researchers found a lot of success in re-using the weights and biases in a multi-layer perceptron in the typical way and appending a classification layer at the very top:

There are many other deep networks which are derived from the RBM. They include Deep Boltzmann Machines, Convolutional Boltzmann Machines, and other Boltzmann machines which use real-valued variables (instead of binary ones). More information about them can be found in Goodfellow's Deep Learning textbook.

Biologically Plausible AI?

These days, most people are aware of VAEs and GANs, i.e., directed generative networks. Although these networks have been met with great success, they lack the biological plausibility and physics-based support that makes RBMs so interesting. Indeed, RBMs started to make a comeback in 2015 by winning the Netflix Challenge, and DBNs have had a surprisingly large number of applications in biomedical engineering. Who knows? Perhaps just as neural networks had a comeback with the advent of more powerful hardware, perhaps Hinton's brainchild will also become more prevalent when we finally achieve biologically plausible artificial intelligence systems!
